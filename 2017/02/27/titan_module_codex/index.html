泰坦系统-开发规范

Posted on 2017-02-27

cedex 法典,记住这个词来源于电影加勒比海盗. 海盗大会中有一步任何海盗都要遵守的条规<海盗法典>
在泰坦这个项目中.我们引入codex作为交互实现的依据,其包括几个层面
1.编码
2.交互
3.数据
4.发布

编码规范
配合eslint完成 对编码的约束,包括变量的命名约定\代码结构\测试覆盖率等

交互规范
codex的核心,面向 RD\UI\PM 三类人群.明确中后台系统的基础交互原则,细节的约定.
包括配色\布局\表单\表格\数据单元的展示原则和交互原则

数据规则
又可以成为接口格式规范,统一接口的约定. 参考GraphQL和RESTfull约定请求模式.
数据格式包括查询请求格式,增改请求格式,数据单元格式.
(数据格式原子化,标准哈)

发布规则
项目发布约定,资源位置,解决何时发布\如何发布的问题

除了基础约定外,我们正在制作一系列配套的工具,完善整个规则体系,使整个规则易于实施.
